LLM — Large Language Model
Large Language Model

Il ne comprend pas —
il complète.

Unsplash

Un modèle entraîné sur des centaines de milliards de mots pour prédire le token suivant. Toute la puissance vient de l'échelle.

Paramètres01
Poids ajustés à l'entraînement. 7B → 1 000B+. Plus = plus capable, plus coûteux.
Fine-tuning02
Ré-entraîner sur vos données métier pour spécialiser le modèle. Touche aux poids.
Inférence03
Phase de production : le modèle génère token par token, en streaming.
Pré-entraînement04
La phase massive — milliers de GPU, semaines de calcul, web entier digéré.
Hallucination⚠ 05
Texte plausible mais faux. Toujours sourcer et vérifier les faits.

Token · Contexte · Température
A — Token

L'unité de traitement.

~¾ mot anglais · 1 000 tokens ≈ 750 mots · Impact direct sur le coût et la latence.

Exemple — 5 tokens
Bon jour le monde !

→ Tokenisation BPE : un mot fréquent = 1 token, un mot rare = 2-5.

B — Context window

La mémoire de travail.

Tout ce que le modèle « voit » en même temps. Prompt + historique + outputs.

GPT-4o128k
Claude 3.5200k
DeepSeek-V3128k
Gemini 1.51M
C — Température

Créativité vs déterminisme.

Échelle 0.0 → 2.0. Contrôle l'aléatoire dans l'échantillonnage.

0.0déterministe
0.7équilibré
1.2créatif
2.0aléatoire

→ Code, SQL, JSON : 0.0–0.3. Écriture : 0.7–1.2.

Skills · System Prompt · Orchestration
Construire un système agentique

Quatre primitives
à connaître.

Skill / Tool

01
Capacité encapsulée donnée à un agent : fonction Python, prompt template, API wrappée. L'agent choisit quel skill invoquer.
→ search_web(query) · send_email(to, subject)

System Prompt

02
Instructions persistantes avant la conversation. Définissent rôle, limites, ton. Invisibles pour l'utilisateur final.
→ « Tu es un assistant RH. Ne parle pas des salaires. »

Orchestrateur

03
Code ou framework qui coordonne plusieurs agents et appels LLM. Gère flux, dépendances, agrégation des résultats.
→ LangChain · LangGraph · CrewAI · n8n · Make

Multi-agent

04
Plusieurs agents spécialisés collaborent. Un manager délègue à des sous-agents : researcher, coder, reviewer…
→ A cherche → B rédige → C valide → D publie

Memory · Grounding · Fine-tuning
Mémoire d'un agent

Quatre étages, des vitesses différentes.

Type Analogie matérielle Description
In-context
RAM		 Volatile · session Tout ce qui tient dans le context window. Rapide, mais effacé à chaque nouvelle conversation. La mémoire la plus immédiate.
External DB
SSD		 Persistant Vector DB ou SQL. Survit entre les sessions. Récupéré à la demande via RAG ou tool use. Le « disque dur » de l'agent.
Épisodique
Journal		 Append-only Logs des actions passées et de leurs résultats. Permet l'apprentissage par l'expérience et le debugging des décisions.
Sémantique
Poids		 Gravé · entraînement Les connaissances dans les poids du modèle. Modifiable uniquement par fine-tuning. La « culture générale » de base.
GroundingRAG + Tools
Ancrer le LLM sur des faits réels. Réduit les hallucinations sans modifier les poids.
Prompt Eng.Instructions
Instructions système + exemples (few-shot) pour guider le comportement, sans toucher aux poids.
Fine-tuningPoids
Ré-entraîner sur vos données. Modifie les poids. Coûteux mais utile pour style spécifique.
